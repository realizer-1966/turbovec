# 문서 7: 임베딩 모델 선택 가이드

임베딩 모델은 텍스트를 고차원 벡터로 변환하는 역할을 합니다. RAG 시스템에서 임베딩 품질이 검색 정확도에 가장 큰 영향을 미칩니다.

nomic-embed-text는 768차원 임베딩을 생성하는 경량 모델입니다. 오픈소스이며 로컬에서 실행할 수 있어 프라이버시가 중요한 환경에 적합합니다.

OpenAI text-embedding-3-small은 1536차원을 생성하며 더 높은 품질을 제공하지만 API 호출이 필요합니다. BGE와 E5 시리즈도 널리 사용되는 오픈소스 임베딩 모델입니다.

임베딩 차원이 높을수록 검색 품질은 향상되지만 메모리 사용량도 증가합니다. turbovec의 4비트 압축은 이 트레이드오프를 완화하여 높은 차원에서도 효율적인 저장을 가능하게 합니다.
